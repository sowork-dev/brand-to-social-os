# Public Roadmap

This roadmap is intentionally simple.
The goal of the first public phase is to make the repo easy to understand, easy to adopt, and easy to extend.

## Phase 1 — Foundation

Status: **in progress**

Goal: make the repo complete enough for a first-time visitor to understand the full workflow.

### Deliverables
- clear positioning workflow for brand → product → social
- starter templates for intake, planning, and review
- basic examples that show how the system works in practice
- public roadmap and issue backlog
- contribution rules and sanitization rules

## Phase 2 — Channel-specific operator packs

Goal: help users apply the system inside specific channels instead of staying at a generic framework level.

### Planned deliverables
- LinkedIn operator pack
- X / Twitter operator pack
- Newsletter repurposing pack
- channel-specific post checklists
- lightweight prompt patterns for different content goals

## Phase 3 — Evaluation and iteration layer

Goal: make the repo useful not just for planning and publishing, but for learning what is working.

### Planned deliverables
- content quality scorecard
- resonance review template
- experiment tracker
- simple weekly review workflow
- examples of message evolution after feedback

## Phase 4 — Industry demo cases

Goal: prove the system can adapt across business types.

### Planned deliverables
- B2B SaaS demo case
- founder-led service business demo case
- agency demo case
- ecommerce demo case

## Prioritization principles

We prioritize work that does at least one of these:

1. makes the repo easier to use without extra explanation
2. helps operators apply the workflow inside a real channel
3. improves proof that positioning can become a repeatable operating system
4. keeps the public version useful without exposing private operating data

## What we are not prioritizing yet

- deep platform automation
- private API integrations
- client-specific playbooks
- enterprise setup complexity
- publishing tools that depend on sensitive credentials
