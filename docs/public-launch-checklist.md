# Public Launch Checklist

## Before posting the repo

- [ ] Re-read the README from a first-time visitor perspective
- [ ] Confirm the repo description matches the README promise
- [ ] Double-check that no private prompts, client names, or credentials were added
- [ ] Make sure each skill has one clear use case and output format
- [ ] Keep the first release intentionally simple

## Suggested launch assets

### 1. GitHub profile / org framing
Use a short one-line description like:

> Open-source operating system for turning brand positioning into product positioning and autonomous social execution.

### 2. Launch post angle
Do not frame this as a generic social media automation repo.
Frame it as:

> We open-sourced the workflow we use to connect strategy → messaging → social execution.

### 3. First demo ideas
- one sample founder workflow
- one before/after positioning example
- one 2-week content plan example

## Good first follow-up releases

1. LinkedIn-specific add-on
2. X/Twitter-specific add-on
3. Newsletter repurposing add-on
4. Lightweight evaluation scorecard for content quality
5. Example operator playbooks for B2B SaaS, agency, and founder brands

## Public messaging rule

The strongest narrative is not “AI writes posts.”
The strongest narrative is “positioning becomes an operating system.”
