# Methodology Overview

## The model

This repo treats marketing as a stack:

1. **Brand layer** — the strategic identity and tension you own
2. **Product layer** — the concrete promise attached to a use case
3. **Distribution layer** — the operating system that turns the promise into repeated market contact
4. **Learning layer** — the feedback loop that sharpens all three

## Why this order matters

Teams often start at layer 3.
They publish before they clarify.
That creates generic content, weak conversion, and endless requests for “more ideas.”

This system reverses the order:

- first clarify what you stand for
- then define what your product means in a buyer's life
- then build recurring content from that position
- then learn from response, not from guesswork

## Outputs at each stage

### Brand positioning
Output:
- strategic audience
- category frame
- alternative map
- tension statement
- differentiation
- point-of-view line

### Product positioning
Output:
- use case
- user job
- pain intensity
- specific promise
- proof stack
- message hierarchy
- objection handling

### Social operating system
Output:
- content pillars
- channel/format matrix
- weekly publishing cadence
- engagement expectations
- repurposing flow
- experiment backlog

### Performance review
Output:
- winning messages
- losing messages
- format lessons
- audience signals
- next iteration priorities
