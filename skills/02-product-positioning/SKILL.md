# Product Positioning

Use this skill after the brand layer is clear.

## Goal
Move from abstract identity to a specific product message that fits a buyer use case.

## Inputs
- brand positioning output
- product or feature name
- user segment
- use case
- alternatives
- proof points

## Workflow

### 1. Define the user job
What is the user trying to get done in one sentence?

### 2. Name the painful alternative
Do not compare against fantasy competitors.
Compare against what people actually do now.

### 3. State the product promise
Use this frame:

> For [user with situation], [product] helps them [desired result] without [main tradeoff].

### 4. Build the message hierarchy
Create:
- primary promise
- 3 supporting claims
- proof for each claim
- common objections and answers

### 5. Turn it into launch language
Write:
- homepage headline
- product one-liner
- short pitch
- 3 activation angles for content and sales

## Output format

```markdown
# Product Positioning

## User and use case
## Job to be done
## Alternative used today
## Product promise
## Why this is different
## Reasons to believe
## Message hierarchy
## Objections
## Launch angles
```
