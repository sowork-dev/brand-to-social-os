# Example Output - Performance Review

> Clearpath. Week 3. LinkedIn. 3 posts. Solo founder.

## Winners

Post: "Your ops team is always the last to know" (Education)
47 likes, 12 comments, 8 saves, 3 DMs.
Why: Hook named a specific feeling. Comments were "this is us every Monday." Audience recognized themselves.

## Losers

Post: "Clearpath connects to 200+ carrier APIs" (Conversion)
9 likes, 0 comments, 0 saves.
Why: Led with a feature. No one cares about API count before they trust you. Wrong pillar timing.

## Unexpected signals

Three commenters mentioned their biggest problem is not exception detection delay but the manual work of notifying the right team member. This is a downstream workflow problem not yet addressed in any content.

## Best comment or objection

"We tried a different tool and it took 4 months to integrate."
Reveals: Strong credibility gap around integration time. The 1-day setup claim needs proof content, not assertion. New angle: "Why integration takes months and ours takes a day."

## Content to double down on

1. The "last to know" frame resonated strongly. Develop 2-3 more posts with this frame across different proof points.
2. The notification workflow problem in comments is a new pillar 1 topic. Add to backlog.

## Experiments for next week

1. If we lead the integration post with "Our customers go live in under a day, not under a quarter" instead of a feature claim, we expect more engagement because it leads with outcome not mechanism.
2. If we post the conversion pillar on Thursday instead of Friday, we expect higher DM response because more people are in planning mode mid-week.
